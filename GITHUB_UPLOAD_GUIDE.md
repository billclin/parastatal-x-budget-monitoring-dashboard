# GitHub Upload Guide

Recommended repository name:

```text
parastatal-x-budget-monitoring-dashboard
```

Recommended visibility:

```text
Public
```

This is safe for portfolio use because the project uses synthetic data only. Keep it private if you later replace any data with real institutional data.

## Option A: Upload using GitHub website

1. Go to GitHub and sign in.
2. Click **New repository**.
3. Repository name: `parastatal-x-budget-monitoring-dashboard`.
4. Description: `Synthetic Kenyan public-sector budget monitoring, forecasting and Power BI analytics project.`
5. Choose **Public**.
6. Do not add a README, .gitignore, or license on GitHub because this package already contains them.
7. Click **Create repository**.
8. Upload all files and folders from this GitHub-ready package.
9. Commit with message: `Initial portfolio project upload`.

## Option B: Upload using Git commands

From inside this project folder, run:

```bash
git init
git add .
git commit -m "Initial portfolio project upload"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/parastatal-x-budget-monitoring-dashboard.git
git push -u origin main
```

Replace `YOUR_USERNAME` with your GitHub username.
